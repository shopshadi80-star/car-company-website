# حالات الاستخدام وقصص المستخدم

# أولًا: حالات الاستخدام الرئيسية

## UC-01: الدخول إلى الصفحة الرئيسية

| البند      | التفاصيل                                                        |
| ---------- | --------------------------------------------------------------- |
| رقم الحالة | UC-01                                                           |
| المستخدم   | زائر الموقع                                                     |
| الهدف      | فهم نشاط المعرض والوصول بسرعة إلى السيارات الجديدة أو المستعملة |
| الأولوية   | عالية                                                           |
| ضمن MVP    | نعم                                                             |

### الوصف

يدخل الزائر إلى الصفحة الرئيسية ويشاهد تعريفًا مختصرًا بالمعرض، مع توجيه واضح إلى قسم السيارات الجديدة وقسم السيارات المستعملة ووسائل التواصل.

### السيناريو الأساسي

1. يفتح الزائر الموقع.
2. تظهر الصفحة الرئيسية.
3. يشاهد تعريفًا مختصرًا بالمعرض.
4. تظهر له أزرار واضحة:

   * سيارات جديدة.
   * سيارات مستعملة.
   * تواصل معنا.
5. يضغط على القسم المناسب.

### معايير القبول

* يظهر تعريف مختصر وواضح بالمعرض.
* تظهر روابط أو أزرار واضحة للسيارات الجديدة والمستعملة.
* تظهر وسائل التواصل الأساسية بوضوح.
* تكون الصفحة مناسبة للجوال.

## UC-02: تصفح السيارات الجديدة

| البند      | التفاصيل                         |
| ---------- | -------------------------------- |
| رقم الحالة | UC-02                            |
| المستخدم   | عميل سيارة جديدة                 |
| الهدف      | استعراض السيارات الجديدة المتاحة |
| الأولوية   | عالية                            |
| ضمن MVP    | نعم                              |

### الوصف

يريد العميل الوصول إلى قائمة السيارات الجديدة المتاحة ومعرفة الخيارات بسرعة ودون تعقيد.

### السيناريو الأساسي

1. يختار العميل قسم السيارات الجديدة.
2. تظهر قائمة السيارات الجديدة.
3. يشاهد صورة السيارة واسمها والموديل والسعر الابتدائي.
4. يضغط على سيارة معينة لعرض تفاصيلها.

### معايير القبول

* يعرض القسم السيارات الجديدة فقط.
* تظهر صورة رئيسية لكل سيارة.
* يظهر اسم السيارة والموديل والسعر الابتدائي.
* لا تظهر السيارات التي أخفاها المسؤول من لوحة الإدارة.

## UC-03: تصفح السيارات المستعملة

| البند      | التفاصيل                           |
| ---------- | ---------------------------------- |
| رقم الحالة | UC-03                              |
| المستخدم   | عميل سيارة مستعملة                 |
| الهدف      | استعراض السيارات المستعملة المتاحة |
| الأولوية   | عالية                              |

### الوصف

يريد العميل تصفح السيارات المستعملة ومعرفة أهم بياناتها من النظرة الأولى، مثل السعر والممشى والسنة والمدينة أو الفرع.

### السيناريو الأساسي

1. يختار العميل قسم السيارات المستعملة.
2. تظهر قائمة السيارات المستعملة.
3. يشاهد لكل سيارة:

   * الصورة.
   * السعر.
   * السنة.
   * الممشى.
   * المدينة أو الفرع.
4. يضغط على السيارة لعرض تفاصيلها.

### معايير القبول

* يعرض القسم السيارات المستعملة فقط.
* يظهر السعر والممشى والسنة بوضوح.
* تظهر المدينة أو الفرع، إن وُجدا.
* تكون المعلومات مختصرة وكافية لاتخاذ قرار أولي.

## UC-04: عرض تفاصيل سيارة جديدة

| البند      | التفاصيل                                      |

| المستخدم   | عميل سيارة جديدة                              |

| الأولوية   | عالية                                         |
| ضمن MVP    | نعم                                           |

### الوصف

يعرض الموقع صفحة تفاصيل موحدة للسيارة الجديدة، تشمل الفئة والموديل والمواصفات الرئيسية والسعر الابتدائي والضمان والخطوة التالية.

### السيناريو الأساسي

1. يفتح العميل صفحة سيارة جديدة.
2. يشاهد صور السيارة.
3. يشاهد الموديل والفئة والمواصفات الرئيسية.
4. يشاهد السعر الابتدائي.
5. يشاهد معلومات الضمان أو الخدمة، إن وُجدت.
6. يضغط على زر حجز تجربة قيادة أو التواصل السريع.

### معايير القبول

* تستخدم جميع السيارات الجديدة قالب تفاصيل ثابتًا.
* يظهر السعر الابتدائي بوضوح.
* تظهر المواصفات الرئيسية بصورة مختصرة.
* يوجد زر واضح لحجز تجربة قيادة.
* يوجد خيار للتواصل السريع.

## UC-06: حجز تجربة قيادة لسيارة جديدة

| البند      | التفاصيل              |
| ---------- | --------------------- |
| رقم الحالة | UC-06                 |
| المستخدم   | عميل سيارة جديدة      |
| الهدف      | إرسال طلب تجربة قيادة |
| الأولوية   | عالية                 |
| ضمن MVP    | نعم                   |

### الوصف

بعد إعجاب العميل بسيارة جديدة، يستطيع إرسال طلب لحجز تجربة قيادة من خلال نموذج واضح.

### السيناريو الأساسي

1. يضغط العميل على زر حجز تجربة قيادة.
2. يظهر نموذج الحجز.
3. يعبئ العميل:

   * الاسم.
   * رقم الجوال.
   * السيارة المطلوبة.
   * الوقت المناسب، إن وُجد.
4. يرسل العميل الطلب.
5. تظهر رسالة تؤكد استلام الطلب.
6. يُحفظ الطلب في لوحة الإدارة.

### معايير القبول

* يحتوي النموذج على الاسم ورقم الجوال والسيارة المطلوبة.
* يرتبط الطلب بالسيارة التي اختارها العميل.
* يُحفظ الطلب في لوحة الإدارة.
* تظهر رسالة نجاح بعد إرسال الطلب.
* لا يُرسل النموذج إذا كانت البيانات الأساسية ناقصة.

* تُحفظ بيانات العميل والطلب.
* تظهر رسالة نجاح للعميل.
* يظهر الطلب ضمن طلبات المعاينة في لوحة الإدارة.

## UC-08: إرسال طلب تواصل عام

| البند      | التفاصيل                     |
| ---------- | ---------------------------- |
| رقم الحالة | UC-08                        |
| المستخدم   | زائر الموقع أو العميل        |

| الأولوية   | متوسطة                       |
| ضمن MVP    | نعم                          |

### الوصف

يمكن للعميل إرسال رسالة عامة إلى المعرض من صفحة التواصل أو من خلال زر التواصل السريع.

### السيناريو الأساسي

1. يدخل العميل إلى صفحة التواصل.
2. يعبئ:

   * الاسم.
   * رقم الجوال.
   * الرسالة.
3. يرسل العميل الطلب.
4. يُحفظ الطلب في لوحة الإدارة.

### معايير القبول

* يُحفظ طلب التواصل العام.
* يحتوي الطلب على بيانات العميل الأساسية.
* يظهر الطلب في لوحة الإدارة ضمن قسم التواصل العام.
* تظهر رسالة نجاح بعد الإرسال.

## UC-09: إدارة السيارات

| البند      | التفاصيل                                    |
| ---------- | ------------------------------------------- |
| رقم الحالة | UC-09                                       |
| المستخدم   | مسؤول الموقع                                |
| الهدف      | إضافة السيارات وتعديلها وإظهارها أو إخفاؤها |
| الأولوية   | عالية                                       |
| ضمن MVP    | نعم                                         |

### الوصف

يستطيع مسؤول الموقع إدارة السيارات الجديدة والمستعملة من لوحة الإدارة دون الحاجة إلى مطور.

### السيناريو الأساسي

1. يدخل المسؤول إلى لوحة الإدارة.
2. يختار إدارة السيارات.
3. يضيف سيارة جديدة أو مستعملة.
4. يعبئ البيانات المطلوبة حسب نوع السيارة.
5. يرفع الصور.
6. يحفظ السيارة.
7. تظهر السيارة في الموقع إذا كانت مفعلة.

### معايير القبول

* يستطيع المسؤول إضافة سيارة جديدة.
* يستطيع المسؤول إضافة سيارة مستعملة.
* تختلف الحقول حسب نوع السيارة.
* يستطيع المسؤول تعديل بيانات السيارة.
* يستطيع المسؤول إظهار السيارة أو إخفاءها.
* لا تظهر السيارة المخفية في واجهة العميل.
